Lightning talks April 2015

Apr 08
43 attended 5/5 by 6 members

After the success of the night last year, this month, we're hosting another night of 5-10 minute lightning talks, this time at Brightpearl!

Leave feedback on Joind.in View on Meetup

Talks

  • Refactoring repositories

    Gavin Davies    , Software Developer at Radify and author of Deal With It: Attitude for Coders

    Sometime a repository begins to smell; it's just not set up right. How can we apply refactoring strategies to source code repositories?

  • OrientDB - The First Days In A Few Minutes

    Richard Jones    , Systems Engineer at Future Publishing

    OrientDB is a graphing database like Neo4j, a document store like Mongo, and you can query it with SQL. This is a brief introduction to getting a server set up, loading it with data and getting that data back out in interesting and fun ways.

  • Drupal 8

    Oliver Davies    , Drupal Developer and a Linux Systems Administrator, working for the Drupal Association

  • The Promised Land

    Warren Seymour    , Software Developer at Radify

    React is an extension that brings event-driven, non-blocking IO to PHP. With this new paradigm comes the 'Promise'; a fascinating and powerful pattern for handling asynchronous behaviour. I'll be using React to demonstrate how Promises rock, and why you should consider them for your own applications.

  • How Much Does Your ORM Cost?

    Stu Herbert    , Freelance PHP / Architect / Quality Assurance at Ganbaro Digital

  • PHP Migrations with Phinx

    Andy Gale    , Devops Consultant at Hello Future

    Automated database migrations are extremely useful when doing Continuous Integration and Delivery. Phinx is a simple PHP standalone tool for writing database migrations for existing or new applications. We like it.

  • Building RPM packages for yum/apt: a guide for the unwilling

    Dan Ackroyd    , Independent C + PHP developer, Imagick maintainer. That guy with the beard

  • The portability problems of ordered hashes

    Tom Adam    , Developer as Instantiate

  • When to declare classes final

    Marco Pivetta    , Software Consultant at Roave, Doctrine core team member & ZF2 contributor

Details
